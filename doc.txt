⚙️ 3. ETL PIPELINE (PRODUCTION LEVEL)
Purpose:
Normalize QuranEnc data
Remove duplicates
Ensure versioning
Bulk insert efficiently
🔥 ETL Rules
✔ Idempotent (VERY IMPORTANT)

Same data run multiple times → no duplication

✔ Batch Processing

1000 records per batch

✔ Upsert logic

Always update if exists

🧪 Production ETL Code
import { TranslationModel } from "../models/Translation.model";

export async function upsertTranslations(batch: any[]) {
  await TranslationModel.bulkWrite(
    batch.map((doc) => ({
      updateOne: {
        filter: {
          surah: doc.surah,
          ayah: doc.ayah,
          language: doc.language,
          edition: doc.edition,
        },
        update: { $set: doc },
        upsert: true,
      },
    }))
  );
}

Avoid full collection scan
Always query by composite index
Use lean queries in Mongoose

⚡ Service Layer Pattern
export class TranslationService {
  async getTranslation(surah: number, ayah: number, lang: string) {
    return TranslationModel.findOne({
      surah,
      ayah,
      language: lang,
    }).lean();
  }
}
⚡ 6. CACHING STRATEGY (INDUSTRY MUST)
Use Redis (Recommended)
Why?
Reduce MongoDB load
Faster response
Scalable system
Flow:
API Request →
  Check Redis →
    HIT → return fast
    MISS →
      MongoDB →
      Store in Redis →
      return
📱 7. OFFLINE-FIRST MOBILE STRATEGY
🔥 Golden Rule:

“Offline is default, online is fallback”

Flow:
App Open →
  Local DB check →
    FOUND → instant load
    NOT FOUND →
      API call →
        save to local DB →
          reuse offline
Local DB Options
Flutter → Hive / Isar
React Native → SQLite
🔄 8. DATA SYNC SYSTEM (VERY IMPORTANT)
